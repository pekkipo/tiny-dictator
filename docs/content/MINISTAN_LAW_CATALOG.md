## Interaction model

Law interactions are data-driven via decision `requirements` (`all_laws` / `any_laws` / `blocked_laws`) and option `add_laws` / `remove_laws`.
Contradictory pairs use `blocked_laws` so both laws cannot enable the same gated card.
Delayed consequences use existing soft/hard follow-up queues on decisions that enact laws.
